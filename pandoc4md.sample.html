pandoc4md.css and markdown2html.sh

What it is all about

Better markdown to html conversion. Better looking HTML. The Gruber script Markdown.pl from here renders HTML without even the <html> nor <body> tags. Bash #!/bin/bash # (I did the shebang to try for the GitHub color rendering.) # using the Gruber script. Markdown.pl Readme.markdown>Readme.html # using pandoc pandoc -f markdown -t html readme.md > readme.html In both cases, the rendered HTML has no headers, no style. It is a partial html file.

How did this happen!!

Many later conversion methods emulate the Gruber script as a starting point. That is why the pandoc conversion gives the same results. Even the all powerful and most wise GitHub markdown to html readme rendering is pretty basic, shhh, quiet.

Is there help out there?

Pandoc is fully pimped out for any conversion needs, a universal document converter get it. Pandoc can give a full and better HTML conversion result; yes the power is ready to burst out into snazzy markdown to HTML renders -- with a css file and a command param.

The local ~/.pandoc directory should be there already with the Pandoc install. So is it there!? Good. Mine is, so that covers all readers right now. But now, to use it.

File ~/.pandoc/pandoc4md.css

The css file I pilfered, edited and renamed pandoc4md.css is right here to help! pandoc4md.css is the beta to the alpha paramater --include-in-header, like this, Bash /usr/local/bin/pandoc --include-in-header="$HOME/.pandoc/pandoc4md.css" -t html -o "$output" "$file" That applies a stylish (if unorthodox) HTML rendering from my tweaked pandoc4md.css. And importantly brings brings the ~./pandoc/ directory, and its stock templates, like ~/.pandoc/default.html to bear in the monumental and, like, cool task of bearable html. Another code example: Bash /usr/local/bin/pandoc --include-in-header="$HOME/.pandoc/pandoc4md.css" -t html "$mdfile" > "$htmlfile" Might render this file like this or not, some browsers show only the source code, not so inspirational.
